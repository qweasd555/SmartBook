# 智记 App 首期联调后端接口文档

## 1. 文档说明

### 1.1 目标
本文档面向 `Spring Boot + MySQL` 后端开发与前端联调，定义“智记 App”首期联调接口协议、鉴权方式、数据结构和业务约束。

### 1.2 当前适用范围
当前版本仅覆盖 App 首期联调需要的 5 个模块：

- 登录模块
- 用户信息模块
- 账单模块
- 共享账单模块
- 账单报表模块

以下模块暂不纳入首期联调范围，后续版本再补：

- OCR 识别
- 预算
- 21 天挑战
- 储蓄目标
- 设置
- 导出
- 分类
- 账本管理

### 1.3 当前联调原则
- 先保证接口路径、参数、返回结构稳定
- 先提供接口骨架与假数据，满足前端联调
- 后续再逐步补齐 service、mapper、数据库与真实业务逻辑
- 文档、Apifox、后端代码三者必须保持一致

### 1.4 环境约定
- 开发环境：`http://localhost:8080/api`
- 测试环境：`https://test.zhiji.com/api`
- 生产环境：`https://api.zhiji.com/api`
- 接口编码：`UTF-8`
- 传输协议：`HTTPS`
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 时区：默认 `Asia/Shanghai`
- 金额单位：`元`

## 2. 通用接口规范

### 2.1 请求头
除登录接口外，默认要求：

```http
Content-Type: application/json
Authorization: Bearer {token}
X-Client-Type: app
X-App-Version: 1.0.0
X-Request-Id: uuid
```

### 2.2 统一返回体

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "2b1f6d2d-7f92-4a4c-a5fd-6f32fd4a1b11",
  "timestamp": "2026-04-10 11:00:00"
}
```

### 2.3 返回码约定
| code | 含义 |
|------|------|
| 0 | 成功 |
| 40001 | 参数错误 |
| 40002 | 业务校验失败 |
| 40101 | 未登录或 token 失效 |
| 40301 | 无权限 |
| 40401 | 资源不存在 |
| 40901 | 重复提交 |
| 50001 | 系统异常 |

### 2.4 分页返回格式

```json
{
  "pageNo": 1,
  "pageSize": 20,
  "total": 156,
  "list": []
}
```

### 2.5 首期联调约定
- 共享账单创建、账单创建建议支持幂等
- 首期阶段允许返回 mock 数据或占位数据
- 前端以文档和 `zhiji-api.json` 作为联调准入标准

## 3. 首期模块划分建议

- 缺少独立的 `auth` 登录模块

- 新增 `module/auth`

## 4. 认证模块

### 4.1 App 登录
`POST /auth/login`

#### 请求参数

```json
{
  "loginType": "WECHAT_APP",
  "code": "wx_app_login_code",
  "mobile": "13800000000",
  "password": "123456",
  "nickname": "小智",
  "avatarUrl": "https://xx.com/avatar.png"
}
```

#### 说明
- `loginType` 首期建议支持：`WECHAT_APP`、`MOBILE_SMS`、`PASSWORD`、`APPLE`
- 首期联调阶段可先以 mock 方式返回 token

#### 响应示例

```json
{
  "token": "jwt_token",
  "refreshToken": "refresh_token",
  "expiresIn": 7200,
  "user": {
    "id": 10001,
    "nickname": "小智",
    "avatarUrl": "https://xx.com/avatar.png",
    "userType": "WORKER",
    "financialScore": 85
  }
}
```

### 4.2 刷新 token
`POST /auth/refresh`

```json
{
  "refreshToken": "refresh_token"
}
```

### 4.3 退出登录
`POST /auth/logout`

## 6. 账单模块

### 6.1 手动创建账单
`POST /transactions`

#### 请求参数

```json
{
  "ledgerId": 20001,
  "type": "EXPENSE",
  "amount": 25.50,
  "categoryId": 502,
  "merchantName": "滴滴出行",
  "occurredAt": "2026-04-08 12:15:00",
  "remark": "打车回家",
  "receiptImageUrl": "",
  "source": "MANUAL"
}
```

### 6.2 编辑账单
`PUT /transactions/{transactionId}`

### 6.3 删除账单
`DELETE /transactions/{transactionId}`

### 6.4 获取账单详情
`GET /transactions/{transactionId}`

### 6.5 分页查询账单列表
`GET /transactions?pageNo=1&pageSize=20&ledgerId=20001&range=TODAY&type=EXPENSE&categoryId=501`

### 6.6 按日期聚合查询账单
`GET /transactions/group-by-date?ledgerId=20001&startDate=2026-04-01&endDate=2026-04-10`

### 6.7 获取最近账单
`GET /transactions/recent?ledgerId=20001&limit=5`

## 7. 共享账单模块

### 7.1 获取共享账单列表
`GET /shared-bills`

### 7.2 创建共享账单
`POST /shared-bills`

```json
{
  "name": "大理旅行",
  "currency": "CNY",
  "members": [
    { "userId": 10001, "name": "我" },
    { "userId": 10002, "name": "张三" },
    { "userId": 10003, "name": "李四" }
  ]
}
```

### 7.3 获取共享账单详情
`GET /shared-bills/{sharedBillId}`

### 7.4 新增分摊记录
`POST /shared-bills/{sharedBillId}/items`

```json
{
  "title": "晚餐聚餐",
  "amount": 356.00,
  "paidByUserId": 10001,
  "splitType": "AVERAGE",
  "participantUserIds": [10001, 10002, 10003, 10004],
  "occurredAt": "2026-04-07 19:00:00"
}
```

### 7.5 获取结算概览
`GET /shared-bills/{sharedBillId}/settlements`

### 7.6 确认结算
`POST /shared-bills/{sharedBillId}/settlements/{settlementId}/confirm`

## 10. 当前结论

从当前项目阶段看，不建议先大幅重构目录结构。更合理的做法是：

- 保留现有项目结构
- 补齐 `auth` 模块
- 统一文档、Apifox、控制层接口
- 先完成 5 个模块的接口骨架与占位实现

只要这条路径跑通，前后端联调就能稳定推进，后面再补真实业务实现即可。
