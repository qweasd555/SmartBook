## 1. 项目概述

### 1.1 产品简介
"智记"是一款面向职场新人、在校学生和家庭用户的智能记账小程序/App，通过OCR拍照识别技术，将繁琐的手动记账简化为"拍照即记"的便捷体验。产品核心解决用户"记账麻烦所以放弃"的痛点，帮助用户建立健康的财务习惯。

### 1.2 产品愿景
让每个普通人都能轻松掌握自己的财务状况，从"月光焦虑"走向"心中有数"的财务自信。

### 1.3 核心价值主张
- **对用户**：3秒完成记账，21天养成习惯，告别月光焦虑
- **对团队**：通过极简MVP验证核心假设，快速迭代扩展场景

## 2. 目标用户画像

### 2.1 核心用户群（优先级排序）

#### A类：月光警戒·职场新人 ⭐ **核心用户**
- **人群特征**：25-30岁，一线城市职场新人，月薪8K-15K
- **核心痛点**：工资到手就光，说不清钱花在哪里
- **使用场景**：每天下班路上拍小票，周末回顾本周消费
- **情感需求**：摆脱月光焦虑，建立财务秩序感

#### B类：家庭财务·统筹主妇/夫
- **人群特征**：35-45岁，管理家庭月度预算
- **核心痛点**：家庭开支模糊，容易超预算
- **使用场景**：每周整理家庭开支，月末做财务报告
- **情感需求**：成为家庭"CFO"，获得掌控感

#### C类：生活费精算·在校学生
- **人群特征**：18-22岁，月生活费1500-3000元
- **核心痛点**：月底余额不足，不好意思开口要钱
- **使用场景**：记录每一笔奶茶/聚餐开销
- **情感需求**：证明自己能够独立管理财务

### 2.2 用户规模预估（首年目标）
- 注册用户：10,000人
- 月活跃用户：3,000人（30% MAU）
- 核心用户留存：1,000人（10% 高活跃）

## 4. 功能需求清单

### 4.1 P0级功能（MVP版本，必须实现）

#### F1：智能拍照记账
**需求描述**：
- 用户打开小程序/App，点击拍照按钮
- 对准小票/收据拍照或从相册选择
- 系统0.5-1秒内识别出金额、商户、时间
- 自动匹配分类，允许用户手动调整
- 点击保存完成记账

**验收标准**：
- 拍照到识别完成时间 < 3秒
- 识别准确率 > 85%
- 支持常见小票格式（超市、餐饮、网购）

#### F2：21天记账挑战
**需求描述**：
- 新用户注册后自动进入21天挑战
- 每天完成记账获得视觉奖励（徽章/进度条）
- 断档无惩罚，但有鼓励文案
- 第7天解锁周账单，第21天解锁消费人格初评

**验收标准**：
- 挑战进度数据持久化
- 每日推送提醒准确送达
- 完成挑战的用户留存率提升50%

#### F3：基础数据管理
**需求描述**：
- 历史记录按时间倒序展示
- 支持按日/周/月筛选查看
- 简单统计：今日支出、本周总额
- 数据本地缓存，网络恢复后同步

**验收标准**：
- 列表加载时间 < 1秒
- 筛选功能准确无误
- 数据同步无丢失

#### F4：周账单报告
**需求描述**：
- 每周日晚自动生成周消费报告
- 展示：总支出、日均、分类饼图
- 对比上周变化（+/-百分比）
- 非必要支出高亮提醒

**验收标准**：
- 报告生成准确率100%
- 图表加载流畅
- 推送准时送达

### 4.2 P1级功能（第2-3个月迭代）

#### F5：家庭共享账本
- 创建家庭空间，邀请家人加入
- 成员权限管理（家长/孩子/只读）
- 家庭公共账户和个人零花钱账户
- 家庭消费看板（本月预算/剩余）

#### F6：校园特供功能
- 校园消费模板（教材费、打印费、社团会费）
- 聚餐AA自动分摊计算
- 匿名校园消费排行榜
- 实习/兼职收支记录

#### F7：消费人格图谱
- 基于3个月数据生成消费人格标签
- 财务健康评分（0-100分）
- 同龄人匿名对比
- 年度回忆账单

### 4.3 P2级功能（第4-6个月探索）

#### F8：发票税务管理
- 增值税发票专用识别
- 税务分类标签（可抵扣/不可抵扣）
- 季度报表导出
- 电子发票自动归集

#### F9：预算规划
- 月度预算设置和跟踪
- 分类预算控制（餐饮不超过XX元）
- 超预算预警提醒
- 预算完成度评分

## 5. 用户流程与场景

### 5.1 核心用户旅程

#### 场景一：职场新人下班记账
```
触发点：下班路上买了杯奶茶
1. 收到推送提醒"今天记账了吗？"（晚8点）
2. 打开小程序/App，点击底部"+"拍照按钮
3. 对准奶茶小票拍照，系统识别"瑞幸咖啡 18元"
4. 自动分类为"餐饮·饮品"，点击保存
5. 看到21天挑战进度条前进一格
6. 周日收到周报告："本周奶茶支出126元，建议控制"
```

#### 场景二：大学生聚餐AA
```
触发点：宿舍聚餐后分摊费用
1. 打开小程序，进入"校园特供"板块
2. 选择"聚餐AA"功能，输入总金额356元
3. 选择参与人员（4人），系统自动计算每人89元
4. 生成收款码分享到宿舍群
5. 收到款项后标记为"已收齐"
6. 记录自动归入"餐饮·聚餐"分类
```

#### 场景三：家庭月度复盘
```
触发点：周日晚上家庭时间
1. 主账号打开"家庭账本"
2. 查看"本月家庭支出报告"：
   - 总支出：8,500元（预算10,000）
   - 最大支出：孩子补习费3,000元
   - 超支分类：外出餐饮（超预算200元）
3. 与家人讨论下月调整方案
4. 设置下月预算：餐饮控制在1,500元以内
```

### 5.2 异常流程处理
1. **OCR识别失败**：提供手动输入界面，记录失败原因用于优化
2. **网络异常**：本地缓存数据，网络恢复后自动同步
3. **用户误操作**：提供撤销功能，删除前二次确认
4. **图片质量差**：引导用户重新拍摄，提供拍摄技巧提示

## 6. 非功能需求

### 6.1 性能要求
- 小程序/App启动时间：< 2秒
- 拍照识别响应时间：< 3秒
- 页面加载时间：< 1秒
- 同时在线用户支持：1,000人

### 6.2 可用性要求
- 新手引导：3步完成首次记账
- 操作一致性：符合微信小程序/App设计规范
- 错误提示：友好明确，提供解决方案
- 无障碍支持：基本色盲友好设计

### 6.3 安全与隐私
- 用户数据加密存储
- 不收集银行卡、身份证等敏感信息
- 图片访问权限控制
- 符合《个人信息保护法》要求

### 6.4 兼容性要求
- 微信版本：支持iOS和Android最新3个版本
- 屏幕适配：主流手机屏幕尺寸
- 网络环境：弱网环境下基本功能可用

## 13. 附录

### 13.1 术语表
- **OCR**：光学字符识别，将图片中的文字转换为可编辑文本
- **MVP**：最小可行产品，用最小成本验证核心假设
- **DAU/MAU**：日活跃用户/月活跃用户
- **NPS**：净推荐值，衡量用户推荐意愿
- **拿铁因子**：看似小的日常开支，长期累积形成大额支出

### 13.2 参考竞品分析
- **网易有钱**：功能全面但复杂，适合深度用户
- **鲨鱼记账**：简单易用但识别功能弱
- **叨叨记账**：强社交属性，聊天机器人记账
- **口袋记账**：传统记账工具，无智能识别

**差异化优势**：专注"拍照即记"的极简体验 + 21天习惯养成
